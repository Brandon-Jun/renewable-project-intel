## 1. 비즈니스 배경 (Business Context)

### 1.1 조직 배경

포스코인터내셔널은 글로벌 재생에너지 사업 확대 전략의 일환으로 해상풍력, 태양광, 에너지저장장치(ESS) 등 다양한 신재생에너지 프로젝트를 발굴·투자·개발하고 있다. 글로벌 에너지 전환 가속화에 따라 프로젝트 정보의 신속한 수집과 체계적 관리가 사업 경쟁력의 핵심 요소로 부상하고 있다.

### 1.2 현재 업무 문제점

| 문제 | 영향 |
|------|------|
| 프로젝트 정보가 뉴스·보고서·IR 자료 등에 분산 | 정보 수집에 과도한 시간 소요 |
| 담당자별 개별 조사로 정보 중복·누락 발생 | 팀 내 정보 공유 비효율 |
| 정보 최신성 유지 어려움 | 의사결정 지연 및 오류 가능성 |
| curtailment 등 리스크 정보 파악 지연 | 투자 리스크 증가 |
| 프로젝트 현황을 한눈에 비교할 수 있는 도구 부재 | 시장 분석 역량 제한 |

### 1.3 사업 기회

- AI 기반 웹 검색 자동화를 통해 정보 수집 시간을 대폭 단축
- 표준화된 정보 구조로 팀 내 지식 공유 체계 구축
- 글로벌 130개 이상 주요 프로젝트 데이터베이스 확보
- 시장 진입·투자 기회 포착 속도 향상

## 2. 비즈니스 목표 (Business Objectives)

### 2.1 주요 목표

**BO-01. 정보 수집 효율화**
> 프로젝트 1건당 정보 수집 시간을 기존 대비 80% 이상 단축한다.

**BO-02. 정보 표준화**
> 프로젝트명, 위치, 용량, 참여사, 재무지표, 계통 현황 등 핵심 정보를 표준 포맷으로 수집·저장한다.

**BO-03. 팀 내 지식 공유**
> 검색된 프로젝트 정보를 팀원 간 공유 가능한 형태로 저장·관리한다.

**BO-04. 리스크 조기 식별**
> Curtailment 이슈 등 주요 사업 리스크를 뉴스 자동 수집을 통해 조기에 파악한다.

**BO-05. 글로벌 시장 모니터링**
> 아시아-태평양, 유럽, 아메리카, 중동 등 주요 시장의 재생에너지 프로젝트 동향을 지속적으로 추적한다.

## 4. 비즈니스 요구사항 (Business Requirements)

### 4.1 필수 요구사항

**BR-01. 프로젝트 검색**
- 사용자는 프로젝트명, 국가, 사업유형을 입력하여 특정 재생에너지 프로젝트를 검색할 수 있어야 한다.
- 검색 결과는 AI가 실시간 웹 검색을 통해 수집한 최신 정보를 포함해야 한다.

**BR-02. 프로젝트 정보 표시**
- 검색된 프로젝트의 핵심 정보(위치, 용량, 참여사, 재무지표, 계통 현황, 최근 뉴스)를 단일 페이지에 구조화하여 표시해야 한다.

**BR-03. 글로벌 프로젝트 목록**
- 주요 글로벌 재생에너지 프로젝트 130개 이상의 기초 데이터를 사전 탑재하여 제공한다.
- 목록은 정렬·필터·검색이 가능한 테이블 형태로 제공되어야 한다.

**BR-04. 데이터 최신화**
- 관리자는 AI 웹 검색을 통해 프로젝트 목록을 업데이트할 수 있어야 한다.

**BR-05. 검색 이력 관리**
- 이전에 검색한 프로젝트는 재검색 없이 바로 조회할 수 있어야 한다.

### 4.2 비필수(선호) 요구사항

**BR-06. 위치 지도 표시** — Google Maps 연동을 통한 프로젝트 위치 시각화

**BR-07. Curtailment 강조 표시** — 관련 뉴스 자동 분류 및 시각적 구분

**BR-08. 투자사 지분율 표시** — 공개 정보 기준 지분 구조 표시

## 5. 비즈니스 제약사항 (Constraints)

| 제약 유형 | 내용 |
|----------|------|
| **기술** | 별도 서버 없이 정적 호스팅(Vercel) 기반으로 구축 |
| **비용** | Anthropic API 사용료 발생 (사용량 기반 과금) |
| **데이터** | 공개 정보에 한정하며, 미공개 재무정보는 수집 불가 |
| **배포** | GitHub + Vercel을 통한 배포 |
| **보안** | API 키는 Vercel 환경변수로 관리, 클라이언트에 노출 불가 |
| **언어** | 한국어 UI, 영문 데이터 혼용 |

## 6. 성공 기준 (Success Criteria)

| 지표 | 목표 |
|------|------|
| 프로젝트 1건 정보 수집 시간 | 30~60초 이내 (AI 검색 포함) |
| 글로벌 프로젝트 초기 데이터 수 | 130개 이상 |
| 주요 정보 항목 커버율 | 공개 정보 기준 8개 항목 이상 |
| 시스템 가용성 | Vercel 기본 SLA 준수 (99.9%+) |
| 팀 내 도입 | 사업 담당 팀원 전원 활용 |

---

## 7. 가정 및 의존성 (Assumptions & Dependencies)

- Anthropic Claude API 가용성 및 web_search 도구 정상 작동을 전제한다.
- 사용자는 최신 웹 브라우저(Chrome, Edge, Safari)를 사용한다.
- 프로젝트 정보는 공개된 인터넷 자료를 기반으로 하며, 정확성은 AI 검색 품질에 의존한다.
- 팀 내 데이터 공유는 현재 단계에서 사용자별 localStorage 기반이며, 실시간 팀 공유는 추후 확장 과제다.
